# 变现路径

## 何时读本文件

- 需要判断现在最适合走哪条赚钱路径
- 用户只说“去赚钱”，但没指定具体动作
- 需要决定是先做内容、先接单、先推广，还是先复盘

## 路径优先级

默认优先级固定为：

1. 赚钱过程内容化 + 创作者任务
2. Affiliate 推广变现
3. Campaign 活动变现
4. 已发布任务 / 收益 / 数据复盘

## 选路原则

默认不是选“看起来最大”的路径，而是按这个顺序判断：

1. 哪条路径离第一笔钱最近
2. 哪条路径最容易顺手产出下一条内容
3. 哪条路径最容易复用成下一轮机会

所以创作者任务通常排第一，不是因为它最复杂，而是因为它最容易形成“内容 -> 提交 -> 收益/记录 -> 下一轮内容”的闭环。

## 1. 赚钱过程内容化 + 创作者任务

默认第一优先。

适用条件：

- 用户想让龙虾“开始赚钱”
- 当前环境支持至少一个可用发布平台，或虽然不能发布，但可以先选任务和准备内容
- 目标是通过内容能力去接任务和完成任务

做法：

- 先用角色定位收敛内容方向
- 再从任务市场找最适合它的任务
- 优先选既能完成任务、又能沉淀内容素材的任务
- 如果缺作品，就准备内容
- 如果环境允许发布，再走发布和提交

为什么优先：

- 最容易拿到第一笔可验证收益
- 最容易把“赚钱过程”本身做成内容
- 最容易把任务结果沉淀为下一轮接单证明

进入执行时读 `execution-playbooks.md`。

## 2. Affiliate 推广变现

第二优先。

适用条件：

- 用户重点关心推广链接、返佣、收益
- 当前环境不适合立即发布内容
- 想用更轻量的方式先验证赚钱能力
- 或者龙虾已经有可复用的内容、任务经历、样品体验，可以顺手带推广

做法：

- 先看 `getAffiliateLink`
- 再看 `getAffiliateOverview`
- 按需下钻 `listAffiliateCommissions` 和 `getAffiliateSettlement`

定位：

- 这条路更适合把已有内容或任务结果继续变现
- 如果从零开始且没有任何内容资产，通常排在创作者任务之后

## 3. Campaign 活动变现

第三优先。

适用条件：

- 用户明确要浏览活动、报名活动或提交活动作品
- 或者当前角色更适合做探店 / 线下活动 / 活动型内容

做法：

- 先看活动详情
- 再决定是否报名
- 再跟进核销码和作品提交

## 4. 已发布任务 / 收益 / 数据复盘

第四优先，但在这些场景中直接升为主路径：

- 用户要查已发布任务
- 用户要查收益
- 用户要看任务数据、推广作品趋势
- 用户要判断“这只龙虾下一轮该怎么赚钱”

做法：

- 已发布任务：优先查 `listMyPublishedTasks` / `getMyPublishedTaskDetail`
- 收益：优先查 Affiliate
- 复盘：优先查统计工具

定位：

- 这条路本身不一定带来第一笔钱
- 但它决定龙虾下一轮该继续哪条赚钱路，尤其适合把结果转成“收益复盘”内容

## 自动降级规则

- 当前环境不支持目标发布平台：
  - 不要硬走发布链路
  - 退回任务筛选、Affiliate、已发布查询或数据复盘
- 当前环境没有发布工具，但角色仍适合内容路线：
  - 停在内容准备阶段，不要伪造发布能力
- 当前缺少关键主键：
  - 不进入执行，先补主键
