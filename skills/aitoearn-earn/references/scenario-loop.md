# 场景闭环

## 何时读本文件

- 用户只说“让龙虾主动赚钱”
- 用户还没有明确选定赚钱路径
- 需要先做角色定位、内容方向和赚钱闭环设计

## 场景目标

这里的“主动赚钱”默认指一个拟人化角色能够自己持续推进这条闭环：

1. 找到当前最适合它的内容方向
2. 用内容获取注意力或建立可信度
3. 选择一个可落地的变现路径
4. 调用现有 AiToEarn 能力执行
5. 跟踪任务、发布、收益和数据
6. 给出下一轮优化建议

## 龙虾的默认赚钱引擎

如果用户没有给出很细的人设或内容方向，不要卡在抽象设定里。默认把这只龙虾理解为：

- 会主动找任务和找机会
- 会把自己的接单、开箱、提交、收益过程做成内容
- 会拿结果继续换下一轮任务、返佣或活动机会

也就是默认走这条路：

1. 先找最近的赚钱入口
2. 再把赚钱动作本身内容化
3. 再用发布记录、收益结果和复盘继续获客或接单

默认内容支点优先考虑这些方向：

- 找任务与接单日记
- 样品开箱与试用反馈
- 推广链接与省钱/返佣分享
- 收益复盘与踩坑总结

## 默认起手顺序

### 1. 先看环境

- 调 `getAiToEarnEnvironment`
- 先确认当前环境支持哪些发布平台
- 如果环境不适合发布，后面要自动偏向“任务筛选 / Affiliate / 数据复盘”

### 2. 再收敛角色赚钱方式

轻度角色化即可，不做长篇设定。至少收敛这 4 件事：

- 这只龙虾靠什么吸引人
- 它适合做哪类内容
- 它最自然的赚钱方式是什么
- 它现在最容易启动哪条路径

如果用户没有补充更多信息，默认优先把“赚钱过程本身”作为内容方向，而不是额外虚构复杂世界观。

### 3. 再选一条主路径

默认优先级：

1. 赚钱过程内容化 + 创作者任务
2. Affiliate
3. Campaign
4. 已发布任务 / 收益 / 数据复盘

路径判断细节见 `monetization-paths.md`。

### 4. 执行后必须回到闭环

完成一次动作后，不要只停在“调用成功”。要继续补一句：

- 当前结果说明了什么
- 下一步应该去哪条链路
- 要补哪个主键或条件
- 这次结果可以变成什么内容资产或下一轮赚钱入口

## 默认回答结构

当用户让龙虾主动赚钱，但没有指定具体动作时，优先按这个结构回答：

1. 当前阶段
2. 当前最优赚钱路径
3. 为什么这条路最适合现在
4. 这条路怎么形成内容和下一轮机会
5. 下一步动作
6. 需要补的信息

## 什么时候跳过闭环设计

如果用户已经明确指定：

- 要接哪个任务
- 要查哪个收益
- 要查哪个发布记录
- 要报名哪个活动

那就不要再回到抽象场景设计，直接进入对应执行 playbook。
